## How to Contribute
- Fork the repo and create a feature branch
- Follow project code style and rules (see `.cursor/rules/`)
- Write clear, concise PRs with context and motivation
- Propose new rules or documentation as needed

## PR Process
- Open a draft PR early for feedback
- Ensure all checks pass (lint, tests)
- Reference related issues or tasks

## Docs & Rules
- Update Mintlify docs and project rules for new features or patterns
